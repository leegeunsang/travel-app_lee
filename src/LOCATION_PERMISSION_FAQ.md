# 위치 권한 FAQ (자주 묻는 질문)

## ❓ "위치 권한이 필요합니다" 메시지가 나타나요

**이것은 정상입니다!** 

브라우저가 당신의 위치 정보를 사용하기 위해서는 당신의 명시적인 허락이 필요합니다. 이는 개인정보 보호를 위한 표준 보안 절차입니다.

### 어떻게 해야 하나요?

**옵션 1: 위치 권한 허용** (추천)
1. 브라우저에서 "허용" 버튼 클릭
2. 자동으로 현재 위치 기반 추천이 시작됩니다

**옵션 2: 수동으로 선택**
- 위치 권한을 허용하고 싶지 않다면 괜찮습니다!
- "인기 여행지" 또는 "지역별 탐색"에서 원하는 지역을 선택하세요
- 모든 기능이 동일하게 작동합니다

## ❓ 콘솔에 에러 메시지가 나타나요

콘솔에 다음과 같은 메시지가 나타날 수 있습니다:
```
Error getting current location region: Error: 위치 정보 접근이 거부되었습니다
```

**걱정하지 마세요! 이것은 에러가 아닙니다.**

- 이는 브라우저가 위치 권한 거부를 알려주는 정상적인 메시지입니다
- 앱은 정상적으로 작동하며, 수동 선택 옵션을 사용할 수 있습니다
- 개발자가 디버깅을 위해 표시하는 정보성 메시지입니다

## ❓ 왜 위치 정보가 필요한가요?

위치 정보는 다음 용도로만 사용됩니다:

1. **현재 지역 파악**: 당신이 어느 지역에 있는지 확인
2. **자동 추천**: 주변 여행지를 자동으로 찾아드림
3. **편의성**: 지역을 일일이 선택할 필요가 없음

### 개인정보 보호

- ✅ 위치 정보는 **서버에 저장되지 않습니다**
- ✅ 오직 현재 세션에서만 사용됩니다
- ✅ 지역명으로 변환된 후 즉시 폐기됩니다
- ✅ 제3자와 공유되지 않습니다

## ❓ 위치 권한을 허용했는데도 작동하지 않아요

### 체크리스트:

1. **기기 위치 서비스 확인**
   - Windows: 설정 > 개인정보 > 위치
   - macOS: 시스템 환경설정 > 보안 및 개인정보 보호 > 위치 서비스
   - Android: 설정 > 위치
   - iOS: 설정 > 개인정보 보호 > 위치 서비스

2. **브라우저 권한 확인**
   - 주소창 왼쪽의 자물쇠 아이콘 클릭
   - 위치 권한이 "허용"으로 설정되어 있는지 확인

3. **네트워크 연결 확인**
   - 인터넷 연결이 활성화되어 있는지 확인
   - VPN을 사용 중이라면 일시적으로 비활성화

4. **페이지 새로고침**
   - F5 또는 Ctrl+R (Cmd+R on Mac)

## ❓ 위치 권한을 거부했는데 다시 허용하고 싶어요

### 브라우저별 재설정 방법:

**Chrome**
1. 주소창 왼쪽 자물쇠 아이콘 클릭
2. "사이트 설정" 선택
3. "위치" 옆의 드롭다운을 "허용"으로 변경
4. 페이지 새로고침

**Firefox**
1. 주소창 왼쪽 자물쇠 아이콘 클릭
2. "권한" 찾기
3. "위치 접근" 옆의 X 클릭 (권한 초기화)
4. 페이지 새로고침하고 다시 허용

**Safari**
1. Safari > 환경설정
2. "웹사이트" 탭 > "위치" 선택
3. 해당 사이트를 "허용"으로 변경

**Edge**
1. 주소창 왼쪽 자물쇠 아이콘 클릭
2. "이 사이트의 권한" 선택
3. "위치" 권한을 "허용"으로 변경

## ❓ 모바일에서도 작동하나요?

네! 모바일 브라우저에서도 완벽하게 작동합니다.

### 모바일 설정:

**iOS Safari**
1. 설정 > Safari > 위치
2. "웹 사이트에 물어보기" 또는 "허용" 선택

**Android Chrome**
1. 주소창 옆 ⓘ 아이콘 탭
2. "권한" > "위치" 허용

## ❓ 개인정보가 걱정돼요

**완전히 이해합니다!** 개인정보 보호는 매우 중요합니다.

### 투명한 정보 공개:

1. **수집되는 정보**
   - GPS 좌표 (위도, 경도)만 일시적으로 사용

2. **사용 목적**
   - 좌표를 지역명으로 변환 (예: "서울", "부산")
   - 해당 지역의 여행지 추천

3. **저장 여부**
   - ❌ 서버에 저장되지 않음
   - ❌ 데이터베이스에 기록되지 않음
   - ❌ 로그에 남지 않음

4. **대안**
   - 위치 권한 없이도 모든 기능 사용 가능
   - 수동으로 지역 선택 가능

## ❓ HTTPS가 필요한가요?

개발 환경 (`localhost`)에서는 HTTP로 작동하지만, 배포된 사이트에서는 HTTPS가 필수입니다.

**이유:**
- 브라우저 보안 정책 (위치 정보는 보안 컨텍스트에서만 접근 가능)
- 사용자 개인정보 보호

## 💡 추천 사항

### 최상의 경험을 위해:

1. ✅ **위치 권한 허용** - 가장 편리한 방법
2. ✅ **GPS 활성화** - 정확한 위치 파악
3. ✅ **WiFi 연결** - 더 빠른 위치 확인
4. ✅ **최신 브라우저 사용** - Chrome, Firefox, Safari, Edge 최신 버전

### 권한 허용이 불편하다면:

- **수동 선택 사용**: 인기 여행지나 지역별 탐색으로 원하는 지역 선택
- **검색 기능 활용**: 검색창에 지역명 입력
- **모든 기능 동일**: 위치 권한 없이도 100% 동일한 추천 받음

   - 페이지 새로고침

3. **다른 브라우저 시도**
   - Chrome, Firefox, Edge 등

4. **문의**

   - 에러 메시지 첨부
